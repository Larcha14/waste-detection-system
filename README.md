# Waste Detection System

Сервис для обработки снимков мониторинговых вылетов и детектирования объектов несанкционированного размещения отходов.  
Система принимает папку со снимками, регистрирует вылет, отслеживает статус обработки изображений, передаёт задачи в очередь сообщений и сохраняет результаты детекции для последующей верификации оператором.

## Services

### flight-service

Сервис отвечает за регистрацию вылетов, хранение информации о снимках и отслеживание статуса обработки.

- `POST /flights`  
  Создаёт новый вылет на обработку: принимает `input_dir`, сканирует папку, создаёт запись в `flights`, создаёт записи в `flight_images`, ставит статус `queued`.

- `GET /flights`  
  Возвращает список всех вылетов: `id`, имя, папка, статус, количество снимков, количество обработанных снимков и количество ошибок. Используется для страницы со списком вылетов.

- `GET /flights/{id}`  
  Возвращает подробную информацию по одному вылету: статус, прогресс обработки, даты создания, старта и завершения.

- `GET /flights/{id}/images`  
  Возвращает все снимки внутри конкретного вылета: `image_id`, имя файла, путь, статус обработки и ошибку, если она возникла.

Статусы обработки вылетов:

- `queued` - Вылет зарегистрирован и ожидает начала обработки;
- `processing` - Снимки вылета находятся в процессе обработки;
- `completed` - Все снимки вылета успешно обработаны;
- `completed_with_errors` - Обработка вылета завершена, но часть снимков обработалась с ошибками;
- `failed` - Обработка вылета завершилась ошибкой;
- `cancelled` - Обработка вылета была отменена пользователем или системой.


- Статусы обработки изображений:

- `queued` - Снимок зарегистрирован и ожидает обработки;
- `processing` - Снимок находится в процессе обработки;
- `completed` - Снимок успешно обработан;
- `failed` - При обработке снимка произошла ошибка;
- `skipped` - Снимок был пропущен и не отправлялся на обработку.

### detection-worker

Сервис отвечает за обработку изображений.  
Получает задачи из RabbitMQ, загружает изображение из хранилища, запускает модель детекции и отправляет событие о завершении обработки.

### detection-results-service

Сервис отвечает за хранение результатов детекции и верификацию обнаружений оператором.

- `GET /images/{id}/detections`  
  Возвращает обнаружения на конкретном изображении: `bbox` или `polygon`, confidence и статус обнаружения.

- `GET /flights/{id}/detections`  
  Возвращает все обнаружения, найденные в рамках конкретного вылета.

- `PATCH /detections/{id}/status`  
  Меняет статус обнаружения: `unverified -> confirmed/rejected`.
  

Статусы обнаружений:

- `unverified` - Обнаружение создано моделью, но ещё не проверено оператором;
- `confirmed` - Оператор подтвердил обнаружение;
- `rejected` - Оператор отклонил обнаружение.


## Data flow

1. Пользователь или внешняя система отправляет запрос `POST /flights`.
2. `flight-service` регистрирует вылет и снимки в базе `flights_db`.
3. `flight-service` публикует задачи на обработку изображений в RabbitMQ.
4. `detection-worker` получает задачи, обрабатывает изображения и публикует события о завершении.
5. `flight-service` обновляет статус обработки снимков и общий прогресс вылета.
6. `detection-results-service` сохраняет результаты детекции в `detections_db`.
7. Оператор просматривает обнаружения и подтверждает или отклоняет их.
